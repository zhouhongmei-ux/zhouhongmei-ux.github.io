Hugo学习笔记

周红梅
Thu, 23 Apr 2020 hugo study

HUGO安装

寻找适合自身系统的版本下载，将下载的压缩包解压，该软件仅有一个hugo.exe，其余两个为README说明文件和LICENSE许可证。以我的安装位置为例：

在C盘中建立一个Hugo文件夹，并在其下建bin和Sites两个文件夹

将压缩包里的hugo.exe放入刚建的C:\Hugo\bin中 将C:\Hugo\bin添加入系统的路径当中。

windows7中添加路径还是有点小麻烦，建议用以下工具

Windows Environment Variables Editor(http://eveditor.com/)

Rapid Environment Editor(https://www.rapidee.com/en/about)

Hugo的使用依赖git系统，在使用hugo前请安装git并建立github账户。

HUGO命令

Hugo命令的风格跟git非常像，基本格式是

hugo [command] [flag]

生成public文件夹

hugo

开启本地服务器

hugo server

此时shell命令行变为不可输入新命令状态，按Ctrl+C结束本地服务器

打开服务器状态之后可以在浏览器中输入localhost:1313预览生成的网站效果

主题关注

主题请关注hugo官网上发布的theme一栏，看了几个大部分是GO语言做的，每一个主题都有相应的使用指导，当然hugo的一些使用技巧还是要看hugo网站提供的doc说明。

Hugo主题

以其中一个叫做Hugo Future Imperfect主题为例https://themes.gohugo.io/future-imperfect/，该页面上链接了这个主题的github仓库，其中也有下载方式说明（英文）

下载该主题只需要在相应文件夹中点右键打开git的bash，用git的clone命令克隆该仓库内容

mkdir themes
cd themes
git clone https://github.com/jpescador/hugo-future-imperfect.git

其实只有第三句是clone命令，第一句建立一个叫themes的文件夹，第二句进入这个文件夹，第三局使用clone命令用https协议克隆目标仓库文件。以前习惯git传输协议的方式，git协议的速度非常快，但最近几次下载git协议方式总是出错，留下资料有空再琢磨。

布置网站

hugo的主题主要还是做个人博客比较适合，建站足够简单，内容丰富，deploy也是非常简单，hugo上有推荐使用netlify或者git page。

Netlify布置网站

netlify的网址是https://app.netlify.com/，进入后需要登录，鉴于前边已经有了github帐号，就以github帐号关联的方式登录，这样也有利于将netlify与github数据仓库关联起来。

netlify上布置已经生成的个人网站极其简单，注意前边提到直接使用hugo命令可以生成一个public文件夹，生成网站的内容都被放在了里边，而你要做的只是把这个文件夹用鼠标拖着拖进netlify页面靠下部的一块虚线方框里，然后安静的等两分钟就可以了。

等deploy过程完成之后会自动生成一个netlify随机生成的域名，但也可以在Domain settings里进行个性化（customize），即Custom domains中进行修改。而这一切过程完全免费并且deploy过程根本不需要自己操心。

另外，netlify可以跟github的仓库关联，当你的仓库中源文件发生变化，netlify这边自动进行更新。
